================================================================================
                        DATABASE SCHEMA (3 TABLES)
================================================================================

restaurants
  id (INT, Primary Key)
  name (VARCHAR 255)
  city (VARCHAR 255)
  created_at (TIMESTAMP)

menu_items
  id (INT, Primary Key)
  restaurant_id (INT, Foreign Key → restaurants)
  name (VARCHAR 255)
  price (DECIMAL 10,2)
  created_at (TIMESTAMP)

orders
  id (INT, Primary Key)
  menu_item_id (INT, Foreign Key → menu_items)
  restaurant_id (INT, Foreign Key → restaurants)
  order_count (INT)
  created_at (TIMESTAMP)

Features:
  ✓ Proper foreign key relationships
  ✓ Cascading deletes for data integrity
  ✓ Indexed foreign keys for performance
  ✓ Timestamp for audit trail

                       GETTING STARTED (6 STEPS)
================================================================================

1. INSTALL DEPENDENCIES (2 minutes)
   cd "Restaurants Dish Search"
   npm install

2. CONFIGURE DATABASE (1 minute)
   Edit .env file with your MySQL credentials:
   DB_HOST=localhost
   DB_USER=root
   DB_PASSWORD=your_password
   DB_NAME=restaurant_db

3. INITIALIZE DATABASE (1 minute)
   node scripts/setup.js
   (Creates database and tables)

4. LOAD SAMPLE DATA (1 minute)
   npm run seed
   (Loads 8 restaurants with 30+ dishes)

5. START SERVER (1 minute)
   npm start
   (Runs on http://localhost:3000)

6. TEST API (< 1 minute)
   curl "http://localhost:3000/search/dishes?name=biryani&minPrice=150&maxPrice=300"
   OR
   http://localhost:3000/search/dishes?name=biryani&minPrice=150&maxPrice=300
